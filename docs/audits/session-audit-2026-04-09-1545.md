## The one-line truth
> v1.1.0 ships a working identifier chain but left two DOI-only guards alive — in `menu.ts` and in pane copy — that directly contradict the feature's own capabilities and will immediately mislead the book and PMID users the release was built to serve.

---

## 🔴 Blockers (all lenses)

1. **[cdo]** `src/modules/citationPane.ts:209` — "No DOI, PubMed ID, or arXiv ID found for this item." ISBN is missing from this message. A researcher whose book has no ISBN sees a message that implies adding one won't help — which is false. Update to: "No recognized identifier found. Add a DOI, PMID, arXiv ID, or ISBN to enable citation data."

2. **[freshen]** `package.json` — `bumpp v9.11.1` carries four high-severity CVEs in its transitive `tar` dependency (path traversal, symlink poisoning, arbitrary file overwrite via hardlink). These fire during every `npm run release`. Fix: `npm install bumpp@^10 esbuild@^0.28.0` and update both ranges in `package.json`.

---

## 🎯 Top 3 highest-leverage fixes

1. **Fix the "no identifier" pane message to include ISBN** `[cdo 🔴 · ux 🟠 · wiring 🟠]` — Single string at `citationPane.ts:209`. The product now supports four identifier types but ships copy that lists three, actively misleading the book-focused researchers v1.1.0 was designed to serve. Also rename the `cg-no-doi` CSS class to `cg-no-identifier` while there.

2. **Replace the DOI guard in `menu.ts` with `extractIdentifier(item) !== null`** `[ux 🟠]` — Currently the right-click "View Citing Works" / "View References" menu items are hidden for any item without a DOI, even when Citegeist has already fetched and displayed citation data via PMID or arXiv. A researcher who sees citation counts in the pane but can't access the network browser from the same item will conclude the feature is broken. This is a one-line fix in `menu.ts` that completes the identifier chain end-to-end.

3. **Bump bumpp to v10 and esbuild to ^0.28.0** `[freshen 🔴🟠 · zeitgeist 🟠]` — Clears four high-severity CVEs that fire on every release, and brings esbuild three minor versions current (including TypeScript 5.x correctness fixes). Command: `npm install bumpp@^10 esbuild@^0.28.0 && npm install`.

## Honorable mention — extract `isBookType` to `utils.ts` `[wiring 🟠]`
`citationColumn.ts:18` and `citationPane.ts:321` both contain identical `item.itemType === "book" || item.itemType === "bookSection"` checks. When `bookChapter` or `encyclopediaArticle` gets added to the suppression list, this will silently drift. Five-minute fix.

## 🌟 Wins worth noting

- **`extractIdentifier()` as a single exported function is the right architecture.** /zeitgeist called the four normalizers "textbook pure-function TypeScript." /wiring noted that ISBN slotted in as step 6 "without disturbing the four steps above it." ISBN, PMID, and arXiv wired into columns, pane, and service with no duplication — the design held.

- **The ISSN_ALIASES + module-load-time merge is exactly right.** 3,177 journals with zero per-lookup cost, e-ISSN dealiasing invisible to callers. /wiring praised it; /zeitgeist confirmed it's consistent with current practice.

- **159 tests, all passing.** The new normalizer suites (normalizePMID, normalizeArxivId, normalizeISBN) have genuine edge-case coverage: ISBN-10 X check digit, old-format arXiv IDs (category/number), PMID with embedded punctuation. /zeitgeist called these out specifically.

- **The Dice coefficient threshold in the DESIGN.md spec (0.92 high, 0.72 medium) was validated by /zeitgeist** as "conservative relative to the bibliometrics literature (typical 0.85–0.90) — appropriate for Citegeist's trust-first stance." The planned v1.2.0 design is sound.
